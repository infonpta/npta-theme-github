## Project Overview

This is a **Shopify theme** for National Personal Training Academy (NPTA), a fitness and nutrition certification training platform. It is based on Shopify's Dawn theme architecture with extensive customization (131 sections, 83 snippets, 53 templates).

The theme is primarily managed via the Shopify admin editor and synced to GitHub — most commits are automated "Update from Shopify" pushes, not manual developer commits.

## Development Commands

```bash
# Start local development server (requires Shopify CLI + store access)
shopify theme dev --store=npta-au.myshopify.com

# Push theme to store
shopify theme push

# Pull latest theme from store
shopify theme pull

# Check theme for errors
shopify theme check
```

There is **no build step** — no package.json, bundler, or transpiler. CSS and JS in `/assets/` are served as-is.

## Architecture

### Page Composition Model

Pages are built through a three-layer composition:

1. **Layout** (`layout/theme.liquid`) — the HTML shell with `<head>`, GTM, analytics, and `{{ content_for_layout }}`
2. **Templates** (`templates/*.json`) — JSON files that declare which sections appear on a page and in what order
3. **Sections** (`sections/*.liquid`) — self-contained blocks with markup, settings schema, and asset references

Sections include **snippets** (`snippets/*.liquid`) for smaller reusable components like product cards, icons, and menu drawers.

### Key Patterns

- **JSON templates**: Templates are `.json` composition files, not Liquid. They reference sections by type name. These files are auto-generated by the Shopify theme editor — edit with care.
- **Section schemas**: Each `.liquid` section ends with a `{% schema %}` JSON block defining its settings, blocks, and presets.
- **Web Components**: JS uses custom elements (`<product-info>`, `<cart-drawer>`, etc.) defined in `/assets/`.
- **Pub/Sub**: Cross-component communication via `pubsub.js` (`subscribe()` / `publish()`).
- **CSS variables**: Theming is driven by CSS custom properties set in `base.css` and overridden per color scheme. Most visual settings flow from `config/settings_schema.json`.
- **Asset loading**: Sections load their own CSS/JS: `{{ 'component-card.css' | asset_url | stylesheet_tag }}`.

### Context-Aware Market Variants

Templates with `.context.ca` suffix (e.g., `index.context.ca.json`) provide Canada-specific page variations without duplicating section code.

### Third-Party Integrations (in theme.liquid)

- Google Tag Manager (GTM-MNW6SCMJ)
- Heatjar analytics
- reCAPTCHA v3 (form submissions in `custom.js`)
- Zapier webhooks (form data handling)
- LottieFiles player (animations)

## Key Files

| File | Purpose |
|------|---------|
| `layout/theme.liquid` | Main HTML wrapper, analytics, global scripts |
| `config/settings_schema.json` | All theme customization options (~1600 lines) |
| `config/settings_data.json` | Current active theme configuration (auto-managed) |
| `assets/base.css` | Global styles, CSS variables, color schemes |
| `assets/global.js` | Core JS utilities and shared functionality |
| `assets/constants.js` | JS constants |
| `assets/pubsub.js` | Event pub/sub system |
| `assets/custom.js` | NPTA custom form handling + reCAPTCHA |
| `sections/header-group.json` | Header composition (nav + popup form) |
| `sections/footer-group.json` | Footer composition |

## Conventions

- Section CSS files follow `component-*.css` or `section-*.css` naming in `/assets/`.
- Snippets are included via `{% render 'snippet-name', param: value %}`.
- Localization uses `{{ 'key.path' | t }}` with translations in `locales/en.default.json` (52 languages supported).
- `settings_data.json` and template JSON files are managed by Shopify admin — avoid manual edits to these unless necessary.
- NPTA-specific sections use descriptive prefixes: `certification-quiz`, `contact-us-blurb`, `elective-section`, `section--process`, `section--tabs-content`.
